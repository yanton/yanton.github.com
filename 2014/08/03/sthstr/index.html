八一八String和StringBuffer

發表於 2014-08-03

在java中有三个类是用来负责字符的操作

  • Char 用来进行单个字符的操作
  • String 对一串字符进行操作，不可改变类
  • StringBuffer 也是对一串字符进行操作，可改变类

今天主要八一八String和StringBuffer的爱恨情仇

固执的String类

我们刚开始接触String的时候，伴随着接触int， double ，boolean等，这会让我们误以为String跟他们一样，也是基础数据类型数据类型。但真相是：String不是原始数据类型，而是引用数据类型。

在这里再复习一下数据类型分类:
数据类型

  • 基础数据类型
    • 数值型
      • 整形(byte,short,int,long)
      • 浮点型(float, double)
    • 字符型(char)
    • 布尔型(true, false)
  • 引用数据类型
    • 类(class)
    • 接口(interface)
    • 数组

String对象两种实例化的区别

  1. 一个字符串就是一个String类的匿名对象，直接赋值实际上就是把一个在堆中开辟好的堆内存空间的使用权给了str对象。使用这个方法的一个好处，就是如果一个字符串已经被一个名称所引用，以后再有相同字符串声明时，就不会再重新开辟空间。
  2. 使用new来声明，所赋的值是一个匿名String对象，然后再开辟一个新的空间，将匿名对象的内容赋给新空间。

字符串内容不可变

内容不可变是String对象的一个特别重要的特性。每次赋值的时候String对象内容改变是通过内存地址的断开-连接变化完成的。而本身的字符串内容并没有任何变化。

但是在开发中，我们会遇到一个字符串需要经常被改变，String类已经无法满足我们了，我们需要祭出大杀器「StringBuffer」

会变通的StringBuffer

StringBuffer支持的方法大部分与String类似。最不一样的一点，就是StringBuffer是可变的字符串，可以使用append()方法来增加字符串。

方法定义 描述
public StringBuffer append(char c) 在StringBuffer中提供了大量的追加操作(与String中使用”+”类似)，可以向StringBuffer中追加内容，此方法可以添加任何数据类型
public StringBuffer append(String str) 添加String类
public StringBuffer append(StringBuffer sb) 添加StringBuffer类

总得有个总结

对于需要频繁修改字符串内容的地方，使用StringBuffer能很大的提高代码的性能。所以在开发中StringBuffer使用较多，这样为了保证用户操作的适应性。
